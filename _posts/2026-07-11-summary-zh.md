## [GitHub 通过迁移到 Unix 风格工具修复了 Copilot 代码审查](https://github.blog/ai-and-ml/github-copilot/better-tools-made-copilot-code-review-worse-heres-how-we-actually-improved-it/) ⭐️ 8.0/10

GitHub 通过将 Copilot 代码审查迁移至共享的 Unix 风格代码探索工具（如 grep 和 find），降低了审查成本并重构了代理工作流，使其专注于拉取请求证据，从而扭转了先前因更好工具导致的性能下降。 这一改进表明，更好的工具并不自动带来更好的 AI 性能，工作流设计至关重要。它为将大语言模型集成到开发者工具中提供了宝贵经验，有望使 AI 辅助代码审查更加高效和可靠。 迁移过程涉及使用标准 Unix 工具在拉取请求上下文中探索代码，从而减少了 token 消耗和成本。该方法强调基于证据的审查，即代理根据实际代码变更验证发现结果，提高了准确性。

rss · GitHub Blog · 7月10日 15:57

**背景**: GitHub Copilot 代码审查是一个自动审查拉取请求的 AI 代理。最初，尝试用更强大的工具增强其功能却意外导致审查质量下降。Unix 风格工具（如 grep 和 find）是标准的命令行实用程序，广泛用于软件开发的文本搜索和过滤。通过采用这些工具，代理能够高效地从代码库中收集证据，从而改善审查效果。
