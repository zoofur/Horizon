## [GigaToken：通过 SIMD 实现 1000 倍更快的 LLM 分词](https://github.com/marcelroed/gigatoken/) ⭐️ 8.0/10

GigaToken 是一个新的开源库，通过利用 SIMD 指令和优化缓存，将语言模型分词速度提升高达 1000 倍，但由于分词通常只占全部推理时间的不到 0.1%，它对整体推理时间的提升有限。 这一突破对需要高吞吐量分词的应用（如数据库查询处理或流式文本处理）具有重要意义，同时它展示了先进的 SIMD 优化技术，可能激发 LLM 流水线中其他部分的类似改进。 GigaToken 的加速来源于使用 SIMD 进行预分词（通常基于正则表达式）、最小化分支以及缓存预分词映射。它支持多种 CPU（x86 和 ARM）和分词器，并提供与 HuggingFace Tokenizers 和 Tiktoken 的兼容模式。

hackernews · syrusakbary · 7月22日 17:20 · [社区讨论](https://news.ycombinator.com/item?id=49010167)

**背景**: 分词是语言模型处理文本的第一步，将原始字符串转换为整数令牌 ID。它通常依赖于基于正则表达式的预分词，这在计算上开销较大。SIMD（单指令多数据流）是一种并行处理技术，允许 CPU 同时对多个数据点执行相同操作，从而显著提高分词等任务的吞吐量。

## [Show HN：Bento – 一个 HTML 文件搞定整个 PPT](https://bento.page/slides/) ⭐️ 8.0/10

Bento 是一个自包含的 HTML 文件（约 560 KB），集成了完整的幻灯片编辑、演示和实时协作功能，无需安装、云登录或加载任何外部依赖。 这种方式简化了幻灯片的创建和分享，尤其适用于希望避免代码编辑的开发者；同时通过加密盲中继提供离线能力和协作隐私保护。 幻灯片数据以纯 JSON 形式存储在文件顶部，应用逻辑则压缩为 base64 编码的 blob，在浏览器中通过 DecompressionStream 解压，从而保持文件小巧且自给自足。

hackernews · starfallg · 7月22日 15:19 · [社区讨论](https://news.ycombinator.com/item?id=49008211)

**背景**: 传统的幻灯片工具如 PowerPoint 或 Google Slides 需要安装或联网。Bento 利用网页技术（reveal.js 和自定义库）创建一个离线可用的独立 HTML 文件。加密盲中继（encrypted blind relay）实现实时协作，且不向服务器暴露数据——中继仅传递加密后的密文。

## [每个人都应了解 SIMD](https://mitchellh.com/writing/everyone-should-know-simd) ⭐️ 7.0/10

Mitchell Hashimoto 的文章认为 SIMD 是重要的性能工具，所有开发者都应了解，并通过具体实例引发了关于其实际适用性和权衡的社区讨论。 了解 SIMD 可以帮助开发者为数据并行任务编写显著更快的代码，影响多媒体处理和科学计算等领域。这场辩论突显了软件优化中的不同优先级，如数据导向设计与微优化之争。 SIMD（单指令多数据）允许 CPU 同时对多个数据点执行相同操作。现代 CPU 包含如 AVX-512 这样的 SIMD 指令集，可以预取缓存行并在单个周期内执行融合乘加操作。

hackernews · WadeGrimridge · 7月22日 17:48 · [社区讨论](https://news.ycombinator.com/item?id=49010648)

**背景**: SIMD 是一种利用数据级并行的并行计算类型，常用于多媒体和科学应用。数据导向设计是一种补充性的优化方法，通过关注数据布局来最大化 CPU 缓存效率，常用于游戏开发。
